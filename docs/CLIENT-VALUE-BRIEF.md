# What This Delivery System Buys You

A one-page, architecture-free statement of value for a client or stakeholder.
You do not need to understand skills, routing, or aliases to read this.

## The problem it solves

AI can generate code quickly. But raw AI output is usually demo-grade: it works
in a screenshot, then a team inherits something they cannot safely change,
deploy, recover, or maintain. The cost lands later - in outages, rewrites, and
"only the person who built it understands it."

This system exists to close that gap. It turns AI assistance into
**production-grade work a team can actually own.**

## What you receive

Not just code. For any meaningful piece of work, the standard deliverable is a
complete package:

| You get | Which means |
|---|---|
| Working software | Built to a defined engineering standard, not improvised |
| Tests with evidence | Proof it works, not a claim that it does |
| A release plan | How it goes live safely, in stages, with go/no-go criteria |
| A rollback plan | How to undo it fast if something goes wrong - written before launch |
| A runbook + named owner | So your team can operate it without calling the author |
| Maintenance notes | What it assumes, what would break it, where to look first |

This package is enforced by a closing checklist (the "Definition of Done"). Work
is not called finished until every item is present.

## The guarantees behind it

| Guarantee | How it is kept honest |
|---|---|
| Consistent quality | Every piece of work runs through the same engineering standard and review gates |
| No silent decay | An automated check runs on every change and fails the build on broken links, duplicates, or oversized or malformed content |
| Domain rigour where it counts | Finance, security, and compliance work carry their own constraints close to the implementation |
| Maintainable handoff | The Definition of Done pack travels with the work, so your team is never stranded |

## Why this is different from "using AI to code"

- **Prompting better** gets you a faster first draft.
- **This system** gets you a first draft *plus* the architecture, tests,
  deployment path, rollback path, and operating manual that make it safe to run
  and cheap to maintain.

The first saves minutes. The second saves the months that bad handoff usually
costs.

## In one sentence

You get software that is not just built, but **proven, shippable, recoverable,
and maintainable** - the standard a serious engineering team would hold itself
to, applied consistently.
