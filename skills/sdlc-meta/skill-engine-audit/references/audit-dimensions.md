# Reference: Audit Dimensions & Output-Type Checklist

Rank EVERY dimension below /100, plus EVERY group and EVERY output type the engine touches. Skip
nothing the engine is responsible for.

## A. Engine dimensions (score each /100)

1. **Doctrine & philosophy** — is there a clear, defensible point of view / standard the skills
   inherit? Is it current?
2. **Taxonomy & structure** — exhaustive? mutually exclusive? balanced (no grab-bag group, no
   2-skill orphans)? Are groups correctly named for what they hold?
3. **Skill depth & rigor** — are workflows deep enough to produce world-class output, or thin
   checklists / boilerplate stubs that re-route to doctrine?
4. **Worked examples & applied proof** — does each craft skill ship a real worked example
   (before/after, applied spec, sample artifact)? (Usually the lowest-scoring dimension.)
5. **Standards currency** — does it encode the CURRENT standards of its field (versions, numbered
   criteria, thresholds)? Or is it stale?
6. **Coverage / output-type readiness** — can it actually drive every deliverable type it's for,
   end-to-end? (See output checklist below.)
7. **Accessibility / inclusivity** — (where relevant) WCAG/criteria, i18n/RTL, etc.
8. **Production / handoff / render fidelity** — tokens, dev-handoff, embedding, file-render — the
   "last mile" to a shippable artifact.
9. **Redundancy & hygiene** — duplicate skills, overlapping scopes, dead/legacy content.
10. **Discovery & routing** — can the right skill be found (clear descriptions, glob-discoverable,
    no stale index)? Does it reference cross-cutting engines correctly?
11. **Safety & integrity** — no unsafe instructions, no hallucination-prone guidance, sourcing
    discipline.

## B. Group scoring
Score each skill group /100 and each skill /100 in a table with a one-line justification. Note
per skill: has reference files? has worked examples? cites doctrine? deep enough or stub?

## C. Output-type readiness checklist (score each that applies /100)

Audit ALL aspects the engine is for — do not stop at one. Typical output types:

- **Marketing website / landing page**
- **Web application** (SaaS dashboard / product UI)
- **iOS app** (HIG / Apple Design Award bar)
- **Android app** (Material 3 / Expressive bar)
- **Cross-platform mobile** (React Native / Flutter)
- **Documents** (report / BRD / SRS as DOCX / PDF)
- **Presentations / decks / pitches**
- **Proposals** (deck + document)
- **Brand / visual identity system**
- **Data product / dashboards / data-viz**
- **Design deliverable / handoff** (tokens, specs, Figma-to-dev)

For non-design engines, swap in that engine's deliverable types (e.g. financial statements,
tax returns, research reports, runbooks, business plans). The rule is the same: **every aspect
the engine produces gets scored.**

For each output type: the score, the 2–3 biggest capability gaps, and which NEW or HARDENED
skills would lift it to world-class. End with a ranked table and an overall readiness score.
