# Monitoring Dashboard Requirements

## Indicator Metadata

Each indicator needs:

- Indicator code and name.
- Definition.
- Result level: output, outcome, or impact contribution.
- Source record.
- Collection method.
- Frequency.
- Disaggregation: country, sex of owner/leader where collected, sector/category, company size, support type.
- Validation rule.
- Owner.
- Reporting format.

## Dashboard Views

1. Pipeline: applications received, eligible, scored, selected, waitlisted, rejected.
2. Geography: applications and selected companies by EAC state.
3. Inclusion: women-led, youth-led, non-capital-city, and other approved categories where consented.
4. Diagnostics: status, scores by domain, binding constraints.
5. TA delivery: package type, expert, sessions, action-plan completion.
6. Results: indicators, trend, target progress, evidence status.
7. Risks/issues: engagement, data gaps, expert availability, compliance issues.
8. Exports: quarterly donor report tables, anonymised public learning summary, internal management view.

## Acceptance Criteria

- Every dashboard number reconciles to a source table.
- Filters cannot expose unauthorised personal or confidential company data.
- Exports are reproducible for a reporting period.
- Missing data is visible and not silently treated as zero.
