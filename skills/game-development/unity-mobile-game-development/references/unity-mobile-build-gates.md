# Unity legacy migration and mobile build gates

Back to [Unity Mobile Game Development](../SKILL.md).

Before build: pin editor/packages; verify Android/iOS dependencies; define dev/QA/release configuration; protect signing; select identifiers, architectures, backend, graphics and stripping from current docs; build through repeatable automation; preserve symbols and manifest.

Device gate: install, cold/warm start, touch, safe area, controller fallback, lifecycle, offline, permission denial, low storage, save migration/corruption, long-session performance, thermal, battery, memory, localisation and accessibility.

Promote the identical tested artefact. Book-era UnityScript/Boo, Web Player, MonoDevelop, `Application.LoadLevel`, legacy input and old Android/iTunes steps are quarantined.
