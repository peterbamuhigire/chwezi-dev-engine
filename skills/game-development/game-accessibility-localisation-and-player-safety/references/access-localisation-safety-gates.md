# Access, localisation, and safety gates

For every critical gameplay action test discovery, activation, timing, precision, hold/repeat, remapping, one-handed/alternate input, feedback redundancy, interruption, and recovery. Record platform/device, input method, assist settings, observed barrier, severity, resolution, and tester basis.

Localisation gates include source freeze/version, context and screenshots, pseudo-localisation, missing/truncated/overflow text, glyph/font coverage, bidirectional layout when applicable, controller prompts, subtitle speaker/timing, voice fallback, culturally sensitive content, and save/network compatibility.

Safety gates cover default communication posture, consent, mute/block/report, rate limits, evidence minimisation, urgent-harm escalation, moderator tooling, consistency review, appeal, transparency, child/guardian controls where applicable, and failure when automated services are unavailable.
