# Narrative Verification Matrix

| Risk | Test | Oracle | Evidence |
|---|---|---|---|
| Broken causality | Beat-table review | Each required beat has explicit cause and state consequence | Reviewed beat map |
| Unreachable content | Automated/manual state traversal | Every required state reachable; invalid transitions rejected | Traversal log |
| Lost critical information | Interrupt/skip/death/save-load cases | Goal and essential fact recover through specified fallback | Build capture |
| False choice | Compare branch state and later acknowledgement | Advertised consequence is observable | State diff/playtest |
| Poor comprehension | Task-based player interview without leading | Project threshold met for goal, stakes and outcome recall | Protocol/results |
| Inaccessible delivery | Channel removal tests | Essential information remains available without audio/colour alone | Accessibility report |
| Cultural harm/error | Named scoped review | Decision, change, dissent and provenance recorded | Review register |
| Localisation overflow | Pseudolocalised build | No clipping; timing and inputs remain usable | Screenshots/log |
| Pacing failure | Representative session playtest | Critical beat duration and abandonment thresholds met | Telemetry + notes |

Define numeric thresholds in the project test strategy; this reference deliberately does not invent universal targets.
