# Polish, balance and milestone gates

Back to [Game Testing and Polish](../SKILL.md).

Rank polish by frequency x visibility x severity x thematic importance, moderated by cost and regression risk. Define done separately for movement/camera, animation, VFX, HUD, audio, writing/localisation, performance and code.

Balance through data: isolate a variable, explore a broad range, narrow it, log build/value/reason/result, then retest dependencies. Telemetry locates a problem; observed play and design intent decide the correction.

External QA begins only with feature-complete internally tested builds, build notes, known issues, environments/accounts and funded fix/retest capacity.
