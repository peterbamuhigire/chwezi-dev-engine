# QA, playtest and bug operations

Back to [Game Testing and Polish](../SKILL.md).

Bug contract: build, device/OS, save/preconditions, exact shortest steps, expected, actual, frequency, severity/player impact, screenshot/video/log, related regression area, owner, fix build and independent verification.

Playtest contract: question, cohort, consent, device/context, task, observation, non-leading interview, metrics, finding confidence and decision. Separate usability, appeal, balance and cultural-review sessions when their questions differ.

Learning tests additionally record construct, age/cohort, comparator, measure, duration and transfer limit. Ad/wellbeing tests record placement/config identity, exposure and decline, action interruption, accidental-tap risk, reward understanding, session exit, regret/trust, complaints/refunds and ability to stop. Do not invent pass thresholds after observing results.

An ad integration bug also records vendor/SDK, audience designation, consent state, creative/ad identifier where lawfully retainable, network/no-fill path, cap/pace state, reward idempotency, close/focus behaviour, save/recovery impact and kill-switch result.
