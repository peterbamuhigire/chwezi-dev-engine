# Mobile design operating playbook

Back to [Mobile Game Design](../SKILL.md).

Rank verbs by frequency and risk. Prototype the most frequent action first, then add one interaction at a time. For each control, record gesture, start zone, ownership, cancellation, conflict, feedback, assistance, handedness, accessibility and failure recovery.

Design five journeys: first 30 seconds, first session, repeat session, interrupted session and failed session. Essential information must survive mute, bright light, hand occlusion, aspect variation, localisation expansion and a player returning after absence.

Use screen flow before visual polish. Give each screen one primary purpose, optimise the common route, test paper wireframes, then verify safe areas and physical-device input. Remove HUD information before shrinking or decorating it.

Do not encode the books' exact three-minute session, historical device lists, acquisition costs or platform rankings. Derive session targets from genre, players and current evidence.
