# Content production gates

Each content unit passes intent, greybox, functional, art/audio, performance, accessibility/localisation, regression, and release gates. Evidence includes owner/version, required assets/systems, navigation and sequence-break tests, checkpoint/save compatibility, streaming budget, instrumentation, playtest question/results, known defects, and approval.

Procedural output requires a retained seed, generator/content version, validity constraints, bounded retry/fallback, reproducible failing sample, and save/network compatibility. Infinite or endless generation claims are rejected unless termination and resource bounds are explicit.

Theresa Hill's *3D Game Development Practical Introduction* is admitted as an OCR-derived, introductory secondary source for the progression from primitive greyboxing through object hierarchies, materials, lighting, cameras, input archetypes, collision/physics, triggers, collectables, puzzles, health, and weapons. Its legacy Unity APIs and uncited performance claims are excluded. The useful production lesson is to test mechanics in small scenes with explicit object relationships before combining them into content; current implementation details require pinned official engine documentation.
