# Profiling and budget playbook

Back to [Mobile Game Performance](../SKILL.md).

Capture build, device, OS, chipset/GPU, RAM, resolution, quality, scene, duration, battery/thermal state, profiler and frame range. Compare CPU main/render/job, GPU, present/frame pacing, allocations/GC, resident/peak memory, I/O/loading, network, audio, physics, AI and animation.

Use identical scenarios and one change per experiment. Keep before/after captures and player-visible outcome. Averages cannot hide hitches; short cold tests cannot prove sustained performance.
