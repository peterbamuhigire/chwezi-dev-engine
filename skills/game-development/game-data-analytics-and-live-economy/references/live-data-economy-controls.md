# Live data and economy controls

Every metric declares purpose, formula, population, time window, exclusions, event/schema versions, data latency, quality thresholds, uncertainty, owner, and decision. Every economy mutation declares source, sink, idempotency key, balance rule, fraud check, refund/reversal path, and audit event.

Remote changes require typed schema validation, environment separation, peer approval, exposure cohort, start/expiry, kill switch, telemetry guardrail, offline/default behaviour, and rollback rehearsal. A dashboard without a decision or a denominator is not evidence.
