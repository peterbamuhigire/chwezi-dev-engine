# Rollout, support and ethical live operations

Back to [Mobile Game Release and Live Operations](../SKILL.md).

Define internal/closed/open/staged exposure, success and halt thresholds, dashboard ownership, support triage, known-issue messaging, rollback decision, hotfix lane and post-launch review. Preserve save and content compatibility across every patch.

For events/DLC, version schedule, eligibility, content, downloads, offline fallback, rewards, economy effects, notifications, localisation, expiry and rollback. Never remove paid access accidentally or turn a cultural narrative into an urgency mechanic.
