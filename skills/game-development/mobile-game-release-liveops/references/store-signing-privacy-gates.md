# Store, signing and privacy gates

Back to [Mobile Game Release and Live Operations](../SKILL.md).

Verify and record dated official requirements for target SDK/OS, AAB/IPA, signing, architectures, privacy manifests/forms, data safety, permissions, age/content ratings, child users, IAP, subscriptions, refunds, restoration, export controls, ads and account deletion.

Release manifest: commit/tag, engine/packages, build config, artifact hashes, symbols, signing identity/custodian, permissions/SDKs, licences/rights, test evidence, store metadata version, rollout and rollback. Never store keys or passwords in it.
