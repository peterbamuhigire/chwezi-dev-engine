# Game trust and enforcement model

Classify controls by purpose: prevent invalid state, detect suspicious patterns, preserve evidence, contain active harm, restore legitimate state, and communicate/appeal. No single client signal justifies irreversible punishment.

For each signal record collection purpose, source, spoofability, retention, access, expected false positives, accessibility conflict, correlation requirements, reviewer, enforcement ceiling, and deletion path. Economy events require idempotency and a ledger-quality audit trail. Security telemetry must itself be authenticated, rate-limited, and monitored for silence or poisoning.
