# Concept-to-engine 3D pipeline

Back to [Game 3D Asset Pipeline](../SKILL.md).

Sequence: verified reference -> thumbnails/silhouette -> turnaround and scale/material callouts -> technical brief -> engine blockout -> topology/normals/transforms -> UV/seam/padding/texel policy -> bake/PBR texture -> rig/skin/extreme poses -> clips/events/root motion -> controlled export -> engine material/collider/LOD/import -> stress-scene acceptance.

Use value, saturation, temperature, shape and detail to guide attention. A concept package must answer downstream production questions, not only look attractive. Maintain one axis/unit/pivot/naming and triangulation contract.
